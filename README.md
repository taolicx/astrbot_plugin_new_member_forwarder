# 新人入群资料转发插件

适用环境：AstrBot + llbot，平台适配器为 aiocqhttp。

## 当前发送流程

新人进群后，插件只走这一条链路：

1. 等待后台设置的入群延迟。
2. 操控本机 QQ，打开左侧置顶的目标群。
3. 在右侧成员栏搜索新人 QQ。
4. 校验资料卡上的 QQ 号和目标一致。
5. 点击“发消息”，发送真人开路文字，默认是“欢迎进群”。
6. 开路成功后，按录制顺序私聊发送新人资料。

真人开路失败时，本次发送直接停止，不计入已发送次数。

真人开路失败时，本次发送直接停止；不会切换到其他发送方式，也不会改变聊天记录的原始卡片样式。

## 录制新人资料

不需要手动找消息编号。

1. 管理员私聊机器人发送：`开始`
2. 继续私聊发送要转发给新人的内容：
   - 文字
   - 图片
   - QQ 合并聊天记录
3. 管理员私聊机器人发送：`结束`

保存后，新人入群时会按录制顺序私聊发送这些内容。

也可以使用更明确的控制词：

- `新人欢迎开始`
- `新人欢迎结束`
- `新人欢迎取消`
- `新人欢迎状态`
- `新人欢迎清空`

## 命令

- `开始`：开始录制新人资料。
- `结束`：保存本次录制，并覆盖旧资料。
- `取消`：放弃本次录制，不覆盖旧资料。
- `状态`：查看是否正在录制、当前录了几条、已保存几条。
- `清空`：清空已保存资料。
- `/新人欢迎测试 QQ号 来源群号`：先执行真人开路，再发送已保存的新人资料；管理员发送、机器人自己发送同名文本都可以触发。
- `/新人欢迎开路测试 QQ号 来源群号`：只执行真人开路，不发送后续资料。
- `/新人欢迎真人开路测试 QQ号 来源群号`：同样只执行真人开路，便于远控测试。
- `/添加一图回复图片`：私聊机器人后直接发送图片，保存为一图回复图片。
- `/添加两图回复图片`：私聊机器人后直接发送图片，保存为两图回复图片。
- `/删除一图回复图片`：删除通过指令添加的一图回复图片。
- `/删除两图回复图片`：删除通过指令添加的两图回复图片。

## 主要设置

- 启用插件：关闭后不处理新人入群和图片回复。
- 管理员 QQ 号：限制谁可以录制新人资料和添加回复图片。
- 生效群号：留空表示所有群都生效。
- 入群后等待秒数：新人进群后多久开始真人开路。
- 每条私信间隔秒数：多条资料之间的发送间隔。
- 重复入群事件去重秒数：避免同一事件被重复处理。
- 启用 QQ 真人开路：开启后操控本机 QQ 打开置顶群并给新人发送第一条文字。
- 真人开路文字：默认“欢迎进群”。
- 目标群置顶行号：目标群在 QQ 左侧会话列表里的行号，第一行填 1。
- 左侧第一行群坐标：当前验证值为 150。
- 右侧搜索结果坐标：当前验证值为 322。
- 单步等待秒数、真人开路总超时秒数：控制 QQ 自动化等待时间。
- 开路成功后等待秒数：开路文字发出后多久开始发录制资料。
- 同一新人开路冷却秒数：短时间重复触发时不重复开路。
- 真人开路调试目录：保存过程截图和结果文件。
- 启用发送次数限制：默认开启。
- 发送次数统计方式：填“按QQ跨群统计”或“按群分别统计”。
- 最多发送次数：默认 2；填 0 或负数表示不限制。
- 录制图片保存到本地：建议开启，避免图片过期。
- 启用私聊一图/两图回复：别人私聊机器人发图时自动回复。
- 一图回复上传图片、两图回复上传图片：后台可直接上传图片，也可以用指令录入。

## 一图/两图私聊回复

- 用户私聊机器人发送 1 张图片后，插件会等待设置的秒数。
- 等待期间没有第二张图，就发送一图回复。
- 等待期间收到第二张图，就立即发送两图回复。
- 管理员正在录制新人资料时，图片会被当作新人资料保存，不触发图片回复。

## 安装

把整个 `astrbot_plugin_new_member_forwarder` 文件夹复制到 AstrBot 的插件目录：

```text
AstrBot/data/plugins/astrbot_plugin_new_member_forwarder
```

然后在 AstrBot 后台重载插件或重启 AstrBot。

## 注意

- 新人入群后只会私聊新人，不会在群里发资料。
- QQ 需要最大化，目标群建议置顶到左侧第一行。
- 右侧成员栏需要能通过 QQ 号搜索到新人。
- QQ 合并聊天记录会按原始聊天记录卡片转发，不拆分、不重建。旧录制内容如果缺少原始转发编号，请重新录制一次。
